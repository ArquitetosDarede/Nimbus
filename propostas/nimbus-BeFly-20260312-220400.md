PROPOSTA TECNICA - MIGRACAO PARA AWS
======================================================================

Cliente: BeFly
Data: 2026-03-12T22:03:50.320862

## 1. Resumo Executivo
A proposta visa a migração da infraestrutura da BeFly da plataforma Azure para a AWS, com foco na transferência de 30 VMs Windows e 10 TB de Blob Storage. Utilizaremos as ferramentas AWS MGN (AWS Application Migration Service) e AWS DataSync para garantir uma migração eficiente e segura. O projeto está previsto para ser concluído em um prazo de 6 semanas, com um orçamento estimado de 50 mil reais. Os benefícios incluem a redução de custos operacionais, aumento da escalabilidade e melhoria na performance dos serviços.

## 2. Contexto e Objetivos do Projeto
A BeFly, uma empresa que opera na nuvem, busca migrar sua infraestrutura atual da Azure para a AWS para aproveitar as vantagens da plataforma, como maior flexibilidade e serviços otimizados. Os objetivos do projeto incluem:
- Migrar 30 VMs Windows Server 2019 e 3 VMs com SQL Server 2022 para a AWS.
- Transferir 10 TB de Blob Storage para a AWS, garantindo que todas as VMs tenham acesso ao armazenamento migrado.
Os critérios de sucesso incluem a migração completa e funcional das VMs e do Blob Storage dentro do prazo e orçamento estabelecidos.

## 3. Requisitos de Negócio
- Migrar 30 VMs Windows para AWS via AWS MGN.
- Migrar Blob Storage de 10 TB para AWS via AWS DataSync.

## 4. Requisitos Técnicos
- Utilizar AWS MGN para migração das VMs.
- Utilizar AWS DataSync para migração do Blob Storage.
- Acesso ao Blob de 10 TB por todas as 30 VMs migradas (leitura e escrita).

## 5. Escopo de Atividades
As atividades do projeto incluem:
- **Planejamento da Migração**: Avaliação da infraestrutura atual e definição do plano de migração.
- **Configuração do AWS MGN**: Configuração do serviço AWS MGN para a migração das 30 VMs Windows Server 2019 e 3 VMs com SQL Server 2022.
- **Configuração do AWS DataSync**: Configuração do AWS DataSync para a transferência do Blob Storage de 10 TB.
- **Execução da Migração**: Realização da migração das VMs e do Blob Storage.
- **Testes Pós-Migração**: Validação do funcionamento das VMs e do acesso ao Blob Storage.
- **Documentação e Treinamento**: Criação de documentação e treinamento para a equipe da BeFly.

## 6. Arquitetura Proposta
A arquitetura proposta para a BeFly na AWS incluirá:
- **Amazon EC2**: Para hospedar as VMs migradas, configuradas com 32 GB de RAM e 16 vCPUs.
- **Amazon S3**: Para armazenar o Blob Storage migrado.
- **AWS MGN**: Para facilitar a migração das VMs.
- **AWS DataSync**: Para a transferência eficiente de dados do Blob Storage.
- **VPC (Virtual Private Cloud)**: Para isolar e proteger a infraestrutura.
- **Sub-redes**: Configuração de sub-redes públicas e privadas para gerenciar o tráfego de rede.

### Descrição do Diagrama de Arquitetura
O diagrama de arquitetura mostrará a interação entre as VMs, o Amazon S3, o AWS MGN e o AWS DataSync, destacando as sub-redes e a VPC.

## 7. Infraestrutura como Código (IaC)
A infraestrutura será provisionada utilizando Terraform. Abaixo está um exemplo de configuração para criar uma instância EC2 e um bucket S3:

```hcl
provider "aws" {
  region = "us-east-1"
}

resource "aws_instance" "windows_vm" {
  ami           = "ami-0abcdef1234567890" # Substituir pelo ID da AMI do Windows Server 2019
  instance_type = "t3.large"
  key_name      = "my-key"

  tags = {
    Name = "Windows-VM"
  }
}

resource "aws_s3_bucket" "blob_storage" {
  bucket = "befly-blob-storage"
  acl    = "private"
}
```
