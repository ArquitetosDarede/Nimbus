PROPOSTA TECNICA - MIGRACAO PARA AWS

Cliente: BeFly

## 1. Resumo Executivo
A proposta visa a migração de 30 VMs Windows e 10 TB de armazenamento Blob da Azure para a AWS, utilizando as melhores práticas de arquitetura em nuvem. O projeto será realizado em um prazo de 6 semanas, com um orçamento estimado de 50 mil reais. A migração proporcionará à BeFly maior escalabilidade, segurança e eficiência operacional, permitindo que a empresa aproveite os recursos avançados da AWS.

## 2. Contexto e Objetivos do Projeto
A BeFly é uma empresa que busca modernizar sua infraestrutura de TI, migrando de uma solução baseada em Azure para a AWS. O objetivo principal é garantir uma transição suave e eficiente, minimizando o tempo de inatividade e maximizando a performance das aplicações. O sucesso do projeto será medido pela conclusão da migração dentro do prazo e orçamento estabelecidos, além da funcionalidade plena das VMs e dados migrados.

## 3. Requisitos de Negócio
- Migrar 30 VMs Windows para AWS.
- Migrar 10 TB de Blob storage para AWS.

## 4. Requisitos Técnicos
- Utilizar AWS MGN (AWS Application Migration Service) para a migração das VMs.
- Utilizar AWS DataSync para a migração do Blob storage.
- Garantir que todas as aplicações funcionem corretamente após a migração.

## 5. Escopo de Atividades
- **Planejamento da Migração**: Análise do ambiente atual e definição da estratégia de migração.
- **Configuração do AWS MGN**: Preparação do ambiente AWS para receber as VMs, incluindo a criação de VPC, sub-redes e grupos de segurança.
- **Configuração do DataSync**: Estabelecimento de uma tarefa de DataSync para transferir os dados do Blob storage.
- **Execução da Migração**: Realização da migração das VMs e dados, monitorando o progresso e resolvendo problemas em tempo real.
- **Testes Pós-Migração**: Verificação da integridade e funcionalidade das VMs e dados migrados.

## 6. Arquitetura Proposta
A arquitetura proposta para a BeFly na AWS incluirá:
- **Amazon EC2**: Para hospedar as VMs migradas.
- **Amazon S3**: Para armazenar os dados migrados do Blob storage.
- **AWS MGN**: Para facilitar a migração das VMs.
- **AWS DataSync**: Para transferir os dados de forma eficiente e segura.
- **Amazon VPC**: Para isolar e proteger a infraestrutura.
- **Amazon CloudWatch**: Para monitoramento e alertas.

### Descrição do Diagrama de Arquitetura
O diagrama de arquitetura mostrará a interação entre os serviços AWS, destacando a VPC, as instâncias EC2, os buckets S3 e as configurações do MGN e DataSync. As conexões de rede e as regras de segurança também serão representadas, garantindo uma visão clara da infraestrutura proposta.

## 7. Infraestrutura como Código (IaC)
A infraestrutura será provisionada utilizando Terraform. Abaixo estão exemplos de código para a criação de uma VPC e uma instância EC2:

```hcl
resource "aws_vpc" "main" {
  cidr_block = "10.0.0.0/16"
}

resource "aws_instance" "windows_vm" {
  ami           = "ami-0abcdef1234567890" # Substituir pelo ID da AMI do Windows
  instance_type = "t2.medium"
  subnet_id     = aws_subnet.main.id

  tags = {
    Name = "Windows-VM"
  }
}
```

## 8. Segurança e Compliance
As medidas de segurança incluirão:
- **IAM**: Controle de acesso rigoroso com políticas de IAM para limitar permissões.
- **Criptografia**: Dados em repouso e em trânsito serão criptografados usando AWS KMS.
- **Auditoria**: Configuração de logs de auditoria com AWS CloudTrail para monitorar atividades.

## 11. Riscos e Mitigações
- **Risco**: Incompatibilidade de aplicações após a migração.
  - **Mitigação**: Realizar testes rigorosos antes da migração final.
- **Risco**: Interrupção do serviço durante a migração.
  - **Mitigação**: Planejar a migração em horários de menor uso e implementar redundância.
